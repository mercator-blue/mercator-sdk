SDK on deck.gl + MapLibre

Ready. Paste your API key and pick a dataset.
Pattern: MapLibre is the basemap host; deck.gl layers are added via MapboxOverlay. Same integration story most customers use today. Flat Mercator only. Globe projection isn't supported when deck.gl layers are hosted by MapboxOverlay — the deck.gl BitmapLayer we extend projects to flat Mercator regardless of the host's projection. Use the Mapbox or MapLibre bindings for globe. Deep-pyramid datasets pan sluggishly. The elevation dataset (maxzoom=8) feels laggy when zoomed past z≈6 because every new tile spawns a fresh BitmapLayer instance with its own shader compile. Shallow pyramids (temp2m maxzoom=5) overzoom one parent and don't hit this. Same caveat as globe — use the Mapbox/MapLibre bindings if smooth panning of high-detail datasets matters.
